## Revisão de código do projeto:
 
- A revisão de código é essencial para um projeto em equipe, ela é responsável por diminuir bugs e garantir um melhor entendimento das ações realizadas no código.
- As alterações passarão por revisão via pull request e só serão integradas à branch principal após aprovação.

**Responsáveis pela revisão**

**Método de escolha do revisor**

**Critérios que serão verificados durante a revisão**

1. Existência de bugs.
2. Código redundante.
3. Falhas de segurança ou privacidade.
4. Códigos mais complexos do que o necessário.
5. Código com code smells
6. Implementação de acordo com os requisitos do projeto.
7. Ausência de testes.
8. Código que viola a arquitetura de camadas (MVC clássico).

**Padrão de comentários na revisão**

1. Evitar comentários subjetivos.
2. Fazer comentários respeitosos.
3. Fazer perguntas e não julgamentos.
4. Sugerir soluções.
5. Justificar os comentários.
6. Comentários restritos ao código.
